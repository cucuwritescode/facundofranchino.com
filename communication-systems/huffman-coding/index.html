Huffman Coding The Art of Efficiency

by Facundo Franchino

Introduction

Standard data storage is often wasteful. When we transmit a message, we typically assign the same fixed cost to every character, regardless of how useful it is. This is inefficient. To save space, we should assign short codes to common letters and longer codes to rare ones.

Huffman coding achieves this by using a frequency-sorted binary tree to encode symbols.

The Huffman Coding Algorithm

$$ \begin{array}{l} \textbf{Algorithm: Huffman Coding} \\ \hline 1. \hspace{0.2cm} \textbf{Input:} \text{List of symbols } S \text{ and probabilities } P. \\ 2. \hspace{0.2cm} \textbf{while } |S| > 1 \textbf{ do} \\ 3. \hspace{0.8cm} \text{Select two symbols } x, y \text{ with lowest probabilities.} \\ 4. \hspace{0.8cm} \text{Create node } z \text{ where } P(z) = P(x) + P(y). \\ 5. \hspace{0.8cm} \text{Assign edge } x \to 0 \text{ and } y \to 1. \\ 6. \hspace{0.8cm} \text{Remove } x, y \text{ from } S. \text{ Add } z \text{ to } S. \\ 7. \hspace{0.2cm} \textbf{end while} \\ 8. \hspace{0.2cm} \textbf{Return} \text{ Tree.} \end{array} $$

Example: Argentina

Let us apply this to the word Argentina. For this demonstration, we treat the letters as case-insensitive.

1. Frequency Analysis

We count the characters in the 9-letter string:

  • $a$: 2
  • $n$: 2
  • $r, g, e, t, i$: 1 each

2. The Comparison

In a standard system (ASCII), every letter costs 8 bits.

$$C_{\text{standard}} = 9 \times 8 = 72 \text{ bits}$$

In our optimised scheme, $a$ and $n$ appear most often, so they get the shortest paths (likely 2 bits). The rarer letters get longer paths (3 or 4 bits).

3. The Result

Summing the bits for the compressed data:

$$C_{\text{huffman}} \approx (2 \times 2) + (2 \times 2) + (5 \times 3.4) \approx 25 \text{ bits}$$

We have reduced the storage requirement from 72 bits to approximately 25 bits. By organising our symbols based on probability, we achieve significantly higher efficiency.
